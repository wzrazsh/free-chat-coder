## 🔍 详细组件对比

### 1. Queue-Server（队列服务器）

**设计文档预期：**
- 目录：`index.js`, `routes/tasks.js`, `websocket/handler.js`, `queue/manager.js`, `evolution/hot-reload.js`, `custom-handler.js`
- 功能：内存任务队列、基础WebSocket通信、简单热重载

**实际实现（大幅超越）：**
```
queue-server/
├── actions/           # 动作引擎（解析、执行、确认）
├── data/              # 持久化存储（tasks.json, evolution-history.json）
├── evolution/         # 自动进化系统（10+个模块）
│   ├── auto-evolve-manager.js    # 进化策略管理
│   ├── self-diagnosis.js         # 自我诊断
│   ├── code-executor.js          # 代码执行器
│   ├── file-executor.js          # 文件执行器
│   ├── evolution-history.js      # 进化历史记录
│   └── hot-reload.js             # 热重载（基础）
├── system-prompt/     # 系统提示模板
└── ...（其余同设计）
```

**核心新增功能：**
- **自动进化系统**：能根据错误类型（WebSocket连接失败、DOM选择器失效等）自动生成修复任务
- **频率限制与风险评估**：进化请求的优先级、冷却期、风险等级控制
- **多轮交互处理**：`custom-handler.js` 中的 `processResult` 支持动作解析与多轮对话
- **持久化存储**：任务和进化历史均保存到JSON文件（原计划迭代3实现）

### 2. Chrome 扩展

**设计文档预期：**
- 简单结构：`background.js`, `offscreen.js`, `content.js`, `manifest.json`
- 功能：维护WebSocket连接、注入脚本、点击发送按钮

**实际实现（模块化架构）：**
```
chromevideo/
├── controllers/       # 控制器层
│   ├── prompt-controller.js     # 提示提交
│   ├── session-controller.js    # 会话管理
│   ├── screenshot-controller.js # 截图控制
│   ├── upload-controller.js     # 附件上传
│   └── mode-controller.js       # 模式切换
├── readers/           # 读取器层
│   ├── chat-reader.js          # 聊天内容解析
│   ├── session-reader.js       # 会话状态读取
│   ├── page-state-reader.js    # 页面状态读取
│   └── model-reader.js         # 模型信息读取
├── utils/             # 工具层
│   ├── anti-detection.js       # 防检测（随机延迟、模拟人类打字）
│   └── dom-helpers.js          # DOM辅助函数
├── auto-evolve-monitor.js      # 自动进化监控
└── ...（其余同设计）
```

**核心新增功能：**
- **模块化分层架构**：MVC-like 结构，职责分离清晰
- **高级防检测**：支持三种打字模式（即时、快速、人类），随机延迟
- **错误自动上报**：DOM选择器失败等错误自动触发进化请求
- **自动进化监控**：性能监控、错误收集、进化触发
