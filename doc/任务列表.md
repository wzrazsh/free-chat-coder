# 任务列表

## P0

### 1. 修复主链路端口不一致
- 优先级：P0
- 目标：让 Web 控制台、Queue-Server、Chrome 扩展使用同一套端口配置，恢复任务闭环。
- 现状：
  - `queue-server` 监听 `8082`
  - `web-console` 代理到 `8082`
  - Chrome 扩展仍连接 `8080`
- 建议动作：
  - 统一 `queue-server`、`web-console`、`chromevideo` 的端口配置
  - 移除扩展中的 `8080` 硬编码
  - 重新验证扩展注册、任务分发、任务回写

### 2. 恢复并稳定 Queue-Server 运行
- 优先级：P0
- 目标：确保 `8082` 服务正常启动并常驻，避免前端持续 `ECONNREFUSED`。
- 现状：
  - `5173` 和 `8081` 可访问
  - `web-console` 日志持续报 `127.0.0.1:8082` 连接失败
- 建议动作：
  - 单独启动 `queue-server`
  - 保留启动日志，定位是否存在启动即退出或异常崩溃
  - 完成 `/health`、`/tasks`、WebSocket 三项连通性检查

## P1

### 3. 收口端口和地址配置
- 优先级：P1
- 目标：减少多处硬编码导致的接线错误和环境漂移。
- 现状：
  - 端口、Web IDE 地址、工作区路径散落在多个模块
- 建议动作：
  - 提取公共配置
  - 统一管理 `web-console`、`queue-server`、`chromevideo` 的连接地址
  - 明确开发环境默认值和覆盖方式

### 4. 验证 code-server 的 Windows 路径修补
- 优先级：P1
- 目标：确认 Web IDE 在 Windows 下能稳定打开目标工作区。
- 现状：
  - `code-server` 已能在 `8081` 启动
  - 最近改动集中在 Windows 路径和 iframe/Web IDE 接入
- 建议动作：
  - 验证 `?folder=` 打开目录是否稳定
  - 验证刷新、重连、切换目录是否正常
  - 验证从控制台跳转到 Web IDE 的流程

### 5. 清理 code-server 运行期错误
- 优先级：P1
- 目标：把 Web IDE 从“能打开”提升到“可稳定使用”。
- 现状：
  - 日志中存在 `vsda` 资源缺失
  - 文件监听存在 `regex_error(error_backref)`
  - 还有请求状态异常日志
- 建议动作：
  - 分类处理资源缺失、文件监听、请求异常
  - 区分阻塞问题与非阻塞噪音
  - 逐项验证是否影响编辑、扩展宿主、文件变更监听

## P2

### 6. 补任务结果展示界面
- 优先级：P2
- 目标：让控制台不仅显示状态，还能直接查看任务结果和错误信息。
- 现状：
  - 当前列表主要展示状态、时间和任务文本
  - `result`、`error` 没有完整呈现
- 建议动作：
  - 增加结果详情区
  - 明确成功结果与失败错误的展示样式
  - 支持查看历史任务回包

### 7. 给 `/evolve` 增加基本防护
- 优先级：P2
- 目标：避免一次错误提交直接把 `queue-server` 改坏。
- 现状：
  - 当前逻辑会直接覆盖 `custom-handler.js` 并重启
  - 没有语法校验、回滚或备份
- 建议动作：
  - 写入前增加基础语法检查
  - 失败时保留旧版本
  - 增加最小回滚方案或临时备份

### 8. 强化 DeepSeek 页面自动化稳定性
- 优先级：P2
- 目标：降低页面结构变化对任务执行的影响。
- 现状：
  - 内容脚本已有模拟输入、随机延迟、轮询回复的 MVP 逻辑
  - 选择器和回复检测仍偏脆弱
- 建议动作：
  - 梳理关键选择器和失败场景
  - 增加更明确的失败分类和上报
  - 验证常见页面状态下的任务执行成功率

## P3

### 9. 收敛工作区噪音和大范围 vendor 改动
- 优先级：P3
- 目标：降低后续排障、评审和回归验证成本。
- 现状：
  - 工作区包含大量 `code-server/lib/vscode/**` 改动
  - 同时存在运行日志文件和业务改动混在一起的情况
- 建议动作：
  - 区分业务层改动、vendor patch、运行日志
  - 清理无价值日志文件
  - 让后续提交边界更清晰
