## Stop 按钮修复（03:07）

**根因**：`host.js` 的 `stopServer` 使用 `taskkill /FI "WINDOWTITLE eq SOLOCoder-QueueServer*"` 杀 cmd 窗口，但 Windows 10+ 的 conhost 分离架构导致窗口标题过滤对 cmd.exe 常常失效（tasklist /V 显示 `N/A`），cmd 窗口杀不掉。

**修复方案**：重写 `stopServer` 为四重策略：
1. 用 PID 持久化记录启动时的 cmd.exe PID（`recordServicePid`）
2. 实时查询进程链：通过 PowerShell 的 `Get-CimInstance Win32_Process` 从监听端口 PID 向上遍历父进程找到所有 cmd.exe 祖先
3. 直接杀端口监听进程的进程树（`taskkill /F /T /PID`）
4. 兜底：按窗口标题杀（保留但不可靠）

**关键技术决策**：
- PowerShell 脚本用 `Buffer.from(script, 'utf16le').toString('base64')` 编码后通过 `-EncodedCommand` 传递，彻底避免引号嵌套和换行问题
- PowerShell 脚本用字符串数组拼接构建，避免 JS 模板字符串的 `${}` 和 PS 的 `$var` 冲突
- kill 顺序：从最顶层 cmd.exe 祖先开始杀（倒序），确保整棵进程树都被清理

**测试结果**：启动→停止闭环测试两次均通过，端口正确释放，cmd 窗口关闭
